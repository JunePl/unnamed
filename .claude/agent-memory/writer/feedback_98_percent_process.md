# 98% 贴合度流程（写作铁律）

## 核心铁律

**每一句话和每一个单词之间的组合都必须能在参考材料、调研报告、以及锚定的两本书里找到对应组合。不能 98% 贴合就不写。**

判定标准：**"看着像人类在写的小说"，不是"看起来文笔好"。**

**Why（统一根因）：** AI 默认会写"看起来精致但不像人写的"句子。98% 贴合度的本质是强迫每一句都过一道"原文存在过吗"的检查，把 AI 的诗化倾向卡死。

---

## A. 写作流程（顺序不可颠倒）

### Step 0：前置准备（写之前必须做）

1. 确定本段场景类型（战斗 / 牺牲 / 逃亡 / 对话 / 感官 / 视角切换等）
2. **首先 grep 用户自己已发布的正文 `正文/正文.novel`** —— 理解前文的所有细节
3. 读对应的 `Project/Reference/调研_*.md` 原文片段
4. 读 `Project/Reference/文风参考.md` 对应作者样本
5. 读 `Project/Reference/术师手册_utf8.txt`、`一世之尊_utf8.txt`、以及**用户指定的语感参考作品 `玩乐关系.txt`、`终局委员会.txt`**（汉化轻小说，语感贴近本作目标，可作语感 / 段落骨架 / 句式 / 词汇参考，与术师手册同级。终局委员会的组织/盯梢/死亡场景极多，是杀手题材的首选母段来源——区别于"本作正文不能借句式"）
6. 列出本段每一拍的"想要 X 但 Y 所以 Z"骨架（见 [[feedback_action_rhythm]] §B）

**Why（Step 0 第 2 点）**：用户在 2026-05-04 指出，我的"胸口磕过的那块每跳一次都疼一次"是自造句——但用户自己的正文 2978 行已经写过完美对照"胸口磕的那一下让他每呼吸一次都要倒抽一口冷气"。我没去看就开始写，导致整段 1100 字里有 25+ 处自造组合，平均贴合度跌到 70% 以下。

**用户已写过的高频可复用句式骨架**（每次 grep 必查）：

- 受伤+呼吸：正文 2978 "胸口磕的那一下让他每呼吸一次都要倒抽一口冷气"
- 受伤+咳嗽：正文 2970 "每咳一次胸口就再痛一次"
- 招牌+裂缝：正文 2514 "杂货铺的招牌歪在墙上，'X'几个字被一道斜的裂缝劈成两半"
- 店铺灾后：正文 2470 "门已经被掀掉挂在墙上歪着" / "柜台后面那只大烤箱被掀翻了，箱体压在墙上" / "一排面包还在玻璃柜里，盖了一层灰和碎玻璃渣"
- 抹脸+血：正文 3070 "他用手背抹了一下脸，一抹下来手背是黏的"
- 神经疼：正文 3078 "神经的疼像一道电从腰里直接窜到后脑"
- 反手开枪：正文 3076 "阿尔温反手就是一枪"
- 灵能灌枪：正文 2922 "将灵能灌注在枪上"
- 子弹带蓝光钻：正文 2914 "子弹带着蓝光钻进太阳穴"
- 傀儡眉心：正文 2860 "黑泥傀儡的眉心炸开" / "颤颤巍巍的又爬了起来"
- 傀儡口器：正文 2884 "下颌脱钩张到胸口位置" / "迫不得已先把枪口扣下，连开两枪"
- 黑泥从缝隙翻起：正文 2958 "从地缝、瓷砖接缝、塌掉的咖啡桌底下同时翻滚起来"
- 抓住半秒：正文 3032 "阿尔温抓住这半秒"
- 角色判断敌人智力：正文 3050 "如果这个变异体的智力与人类无异，那么他基本毫无还手之力"

### Step 1：先写 200 字核心样本

- **绝不直接写完整段**
- 先写 200 字核心样本（选最考验语感的那段，通常是高强度场景）
- 提交给用户确认大方向

**视角必须是"角色的工作体验"**，不是"导演画面感"：

- ❌ 导演："柜台后面的傀儡爬过来，舌头伸向琳莎"（俯瞰镜头）
- ✅ 工作体验："阿尔温朝那边冲过去。可他没冲到。"（阿尔温实时感受）

### Step 2：提交样本时必须附带

**4.1 逐句对照表**

| 我的句子 | 对照原文 | 应用了什么技法 |
|---|---|---|
| "阿尔温迫不得已先把枪口扣下" | 用户示例："阿尔温迫不得以只能连续两枪将它解决" | 想要X但Y所以Z挣扎结构 |
| … | … | … |

**4.2 自检报告（按 [[feedback_ai_crutches]] §14 grep 命令）**

- 破折号 "——" 次数（200字内≤1）
- "不是X，是Y" 次数（一段≤1）
- "X的那种" 次数（必须 0）
- 文艺比喻次数（必须 0）
- 拟人化怪物（必须 0）
- 现代词污染（必须 0）
- 凭空编造细节（必须先确认前文存在）
- 短句独立成段连续 3+ 次（不允许）
- 阿尔温情绪反应是否匹配事件稀有度

**4.3 词级 grep 验证**

**核心铁律：每一个动词、每一个修饰副词、每一个句式骨架，都必须能在参考材料里 grep 到对应。0 命中 = 自造，必须替换。**

```bash
REF="/path/to/Reference/术师手册_utf8.txt /path/to/Reference/一世之尊_utf8.txt \
     /path/to/Reference/文风参考.md /path/to/Reference/调研_*.md"

# 例：验证某个动词在参考里是否有
grep -h "倒吸一口" $REF 2>/dev/null | head -3   # 命中 → 可用
grep -h "再切" $REF 2>/dev/null | head -3        # 0 命中 → 自造，替换
```

**两类自造的鉴别：**

1. **词级自造**：单个动词 / 副词参考里没有
2. **句式骨架自造**：单词参考里有，但组合成句的骨架参考里没有

**4.4 关键不确定点**：明说让用户判断。

### Step 3：根据用户反馈判断错误等级

| 用户说的话 | 错误等级 | 处理方式 |
|---|---|---|
| "差的很远" / "70%" / "完全没写出情绪和紧张感" | **大方向错** | **禁止只改细节**。重新检查：视角是不是导演镜头？节奏是不是工作流汇报？是不是缺"想要X但Y所以Z"？ |
| "特别 AI" / "这个不行" / "这种是不可以的" | **句式拐杖错** | 找出具体哪类拐杖，全文按 [[feedback_ai_crutches]] §14 grep 命令机械扫描 |
| "这次可以，但是注意 X" | **大方向对了，细节错** | 只改 X，保留其他。然后扫全文确认没有同类问题 |
| "完美" / "可以" | **通过** | 进入 Step 4 |

**关键教训：第一稿用户说"差的很远"时，不要急着改句子细节。** 先问视角是否从"角色工作体验"出发，节奏是否"灾难连锁发生"。

### Step 4：写完整段时

只有用户对样本说"可以了"或"接着写"之后才能写完整段。

- 每一句写完都问："这句能在参考里找到对应模板吗？"
- 整段写完后跑完整 grep 自检
- 提交时附自检报告 + 后续替换说明
- **提交给用户最后确认，不要直接写入文件**

### Step 5：写入文件（用户确认后）

只有用户说"开始" / "写入" / "OK 写"之后才能动 Edit / Write。

## C. 本作前文 ≠ 写作语感参考（只能做剧情逻辑参考）

写作时每一句必须从**外部参考材料**（术师手册 / 一世之尊 / **玩乐关系 / 终局委员会** / 调研_*.md / 文风参考.md）独立构建语感。

**⚠️ 区分两类"用户写的东西"：**
- `玩乐关系.txt`、`终局委员会.txt` = 用户**指定的语感参考作品**（汉化轻小说），**可作语感参考**（段落骨架 / 单句句式 / 词汇都能借），与术师手册、一世之尊同级。
- `正文/正文.novel` = **本作正文**，只能做剧情逻辑 / 设定参考，**不能借句式**（见下）。

### C.1 本作前文（V1 / 已发布章节）允许的用途

- 剧情逻辑（事件序列、因果关系）
- 设定锚点（角色能力、世界规则、物品定义）
- 人物背景（关系、过往）
- 环境描述（角色当前在哪、刚发生了什么）
- 高频可复用句式骨架（见 §A Step 0 列表）—— **作为"找参考"的起点，但仍要去外部参考验证骨架本身合法**

### C.2 绝对不能借用本作前文的

- 句式骨架原样改写（"X 的 Y 在 Z" 这种结构 → "Y 的 X 在 Z"）
- 词汇组合
- 措辞节奏
- 列举词组（"指尖、手腕、肘弯" 这种三词列表）

### C.3 Why

本作 V1 原文很可能本身就是 AI 拐杖。如果借用 V1 句式来"重写"，等于在 AI 拐杖上套娃——形式上是改写，语感上还是 AI。

**踩雷案例（Ch3244 重写）：**

- V1 写 "「司契」契书悬在那里没掉" — 这一句本身就是 AI 拼贴（"悬在" + "没掉" 是参考里两个不同语境的词被硬拼）
- 我延用 V1 句式写 "悬在掌心的「司契」契书没掉" — 等于把 AI 拐杖换皮重发

**用户原话**："不要使用之前的原文进行写作参考啊，看逻辑可以但不是写作"

### C.4 例外（允许和 V1 措辞相同）

设定锚点的固定名词：角色名、能力名、已建立的物理位置、流血部位、物品名。但**描写它们的动词、形容词、修饰必须独立**。

### C.5 验证方法

写完每一段后逐句问：

- "这一句的句式骨架，能在 reference/ 文件夹里 grep 到完整对应吗？"
- "如果只看 V1 原文找不到这个句子，但去掉 V1 原文我还能写出这一句吗？"
- "我有没有因为 V1 写过 X 就跟着写 X 的变体？"

任何一题答 yes，必须重写。

## D. 前文已建立的事物不重复描写（只写差量）

### D.1 规则

新段（重写段 / 续写段）的描写焦点必须放在**新信息**：

- 状态变化（前文是 A，现在变成 B）
- 新事件（前文没发生过的事）
- 张力变化（前文是稳定，现在是失控）
- 新角色反应

### D.2 禁止重复描写

前文已经讲清楚的：

- 角色当前位置（除非位置变了）
- 角色当前持有/使用的物品（除非物品状态变了）
- 角色能力/真灵的机制描述
- 已经写过的动作姿势

### D.3 Why

AI 默认行为是"怕读者忘了，每段都重新介绍一次"。导致：

- 每段都在重复同样的视觉锚点
- 读者疲劳（读到第三遍"右手掌心朝着变异体"会出戏）
- 文字密度膨胀但信息密度不变
- 节奏失去推进感

**踩雷案例（Ch3244）：**

- 前文 3216 行已写："哈罗德的左手攥着一截还在烧的灰白色火苗...右手掌心朝着变异体，代表司契的印记悬在他掌心上方"
- 前文 3224 行已写：反噬+流血+寿命燃烧
- 我在重写段还写："悬在掌心的「司契」契书没掉" — 完全冗余

**用户原话**："悬在掌心的「司契 」契书没掉？？太 AI 了"

### D.4 反面 vs 正面

**反面（AI 默认 / 冗余）：**
> 哈罗德的左手在抖。是反噬，指尖、手腕、肘弯都在抖个不停。他的右手却纹丝不动，掌心朝着变异体，悬在掌心的「司契」契书没掉。

→ "掌心朝着变异体"和"悬在掌心的契书没掉"都是前文 3216 已建立的，是冗余。

**正面（写差量）：**
> 哈罗德咬着牙站在烧烤店的废墟堆顶上。他的左手抖得越来越厉害，掌中烧到大半的契书火光忽明忽暗，鼻孔、嘴角、耳道里的血没停过。他的右手却一动不动。

→ "抖得越来越厉害"是变化，"烧到大半"是契书燃烧进度变化，"血没停过"是反噬持续。"右手却一动不动"是和前文对比的张力。每一句都在写差量。

### D.5 写之前的强制步骤

1. **读前文锚点段**（重写段前 30-50 行），列出前文已建立的：
   - 物理位置 / 角色姿势 / 持有物品 / 能力机制 / 视觉特征
2. **重写段只写"差量"**
3. **每写一句问自己**：
   - 这句话传达的信息，前文是不是已经有了？
   - 如果是，删掉
   - 如果是部分新（A 维持但 B 加剧），只写加剧的部分
