# 节奏→骨架→填词三段法

**Why（用户原话）：**
"开始先分析如何构建片段的节奏 → 每句话应该使用什么结构（使用grep对参考作品进行骨架结构搭建，骨架要保证100%贴合）→ 开始写（使用grep对参考作品进行用语/词语搭建，确保没有自造词）→ 写完之后用另外一个 skill 进行验证"

**Why（我的根本问题）：** 之前我"先想内容再凭感觉填句子"——句式骨架是我凭感觉写的，所以经常出 AI 默认偏好的句式（短独段堆砌/破折号过多/cliché 比喻）。**这次新工作流强制把"骨架"从"凭感觉"变成"参考验证"**。

---

## ⭐⭐⭐ 第〇铁律：段落骨架（最高优先级，高于本文件其余一切）

**Why（用户原话 2026-06）：**
"我们对于骨架的注重严重不够，必须对 agent 的提示词做出改变，让 agent 对于骨架的注意达到最高。注意我们说的骨架是指**一整句话/一整段落**的骨架。"

**根本认知纠正：** 骨架 ≠ 单句句式（"X 的人 Y"那种）。骨架 = **一整段的「情绪推进逻辑 + 修辞手法组合顺序」**。
**即使每个单句都 grep 过，只要整段的组织逻辑是 AI 自己拼的，写出来还是 AI 味。** 段落骨架是防 AI 味的最高一级，必须先于单句句式、先于词级。

### 操作：整段从真实段落抽取骨架，整段套用

1. **定效果**：这一段要达到什么效果？（吐槽/落差搞笑/紧张/悲凉/神秘/克制/……）
2. **找母段**：去参考作品（术师手册 / 一世之尊 / **玩乐关系 / 终局委员会** / 文风参考）里找一个**达到相同效果的真实段落**（一整段，不是单句）。玩乐关系、终局委员会是用户指定的语感参考作品（汉化轻小说），最贴本作目标语感；终局委员会尤其多组织/盯梢/死亡场景。
3. **抽骨架**：拆出母段的 = 情绪推进逻辑 + 修辞手法的**组合顺序**。
4. **套写**：用**完全相同的段落骨架**，填入本作的内容。

### 示范（用户原例）

参考母段（术师手册）：
> 可笑的是亚修刚才还以为这是自己穿越外挂，没想到是人手一份的大众货。简直就是古代人穿越到现代，捡到一部手机以为是神器。

抽取的段落骨架：
- 【前后落差自嘲】可笑的是，X 还以为是 A（自以为独有/厉害），没想到是 B（其实烂大街/没用）
- 【夸张类比收尾】简直就是（一个具体、好笑的场景 C）

同骨架套用本作：
> 阿尔温本来还以为捡到了免费大便宜，没想到是这种鸡肋能力。简直就是【一个属于阿尔温生活记忆库的好笑比喻】。

### 铁律

1. **写每一段之前，先找到那段的"参考母段"**——一个真实存在、达到相同效果的整段。
2. **把母段骨架（情绪逻辑+修辞顺序）写进规划给用户看**："这段用的是 X 作品 Y 处的段落骨架：【…】"。
3. **没有母段 = 不能写。** 凭空组织一个段落的推进逻辑 = AI 味的总根源。
4. 优先级：**段落骨架 ＞ 单句句式（A.2）＞ 词级（A.3）**。三层都做，但段落骨架是第一位的。
5. **grep 验证语法警告**：`grep -E` 里"或"是 `|` 不是 `\|`。写 `\|` 会被当字面竖线，造成大量**假 0 命中**（"盯着""落得"等其实有参考却报 0）。验证骨架务必用 `|` 或单词单查。

### 执行细则（2026-06 重写第18章·用户逐条调教出来的，全是血泪）

1. **⭐⭐⭐ 片段级零填充（最严标准·最高优先级·2026-06 用户原话确立）**：成稿**不允许任何"自己写、没出处"的部分**。
   - 用户原话："你可以在句子本身一直 iterate，但不允许自己填充，比如画面长句里面有一段不适用，那么这一小段要从其他的地方继续找母句直到圆满。"
   - **每个片段（短语级，小到几个字）都必须能在参考里 grep 到母句出处**；一句话可由多个母段的片段拼接而成，但**每个片段都要有出处**。
   - 某片段找不到 → 换说法 / 继续从别处找母句，直到整句每个片段都圆满（iterate，可在句子上反复打磨）。
   - 连接词/虚词/断句/标点全部保留，只在母段骨架的空位上填实义词；母段几句→成稿几句，结果像"母段的填空版"（亚修示范：还以为→没想到是→简直就是，一字不动）。
   - **成稿时必须把每个片段的【片段 → 母句出处】实际列成表自我监督**。最爱犯的偷工＝"标几个母段当幌子，段里大半句子其实自由写"（约纳斯死亡段 13 句只 4 句有母段就是铁证），用户一眼看穿。
   - grep 片段用 `grep -hoE "[^，。」「]{0,4}片段[^，。」「]{0,6}"` 看它在参考里真实存在的上下文。
2. **实义词也必须从参考取，且符合目标语感**：骨架对了但实义词出戏照样废。江湖老派/侦探腔词（盯梢、不挪窝、做掉、手脚利索、节外生枝、藏污纳垢、够本、大活人、按说）在轻小说参考里 0 命中＝出戏。每个实义词都要在**同语感参考**里 grep 到。
3. **每一段都要填母段，不能只做开头几段就放飞**：最爱犯的偷工——前两段认真填母段，后面一大段又凭感觉自由写+事后只 grep 实义词。用户一眼看穿。**成稿时必须把每段的【母段+成稿+对照+验证】都实际列出来**，自我监督。
4. **上位替代，不流水账**：剧情确定后别用第一反应的平铺直叙（"他走到底，转身，靠墙站好，等"），去参考找更高级的写法（母段就是上位替代的来源）。
5. **从具体细节切入，不概括陈述**：开篇/段落别用"X做了十二年、从没见过"这种概括，从一个具体、带世界钩子的细节切入（造梦仪广告→借角色反应立人性→把职业藏进动作→职业背景最后才带出）。
6. **母段"同构"指修辞/情绪类型同构，不是剧情场景同构**：杀手死巷题材在参考里没有同场景母段，但"被逼入绝境""猝死瞬间""临终念头"等修辞/情绪类型的母段到处都有，套其骨架即可。
7. **参考池：只用三本轻小说（2026-06 用户指令，停用一世之尊）**：**终局委员会**（黑帮/组织/盯梢/死亡）、**玩乐关系**（日常/吐槽/喜剧）、**术师手册**（战斗/危险/对峙）。按要写的情绪类型选库。
   - ⛔ **一世之尊（仙侠白话）已停用作母段池**——它的语感会把轻小说味稀释成仙侠味（典型污染词：运起/古井无波/压箱底/催动/灵光/孟奇/真灵气息）。只可做剧情逻辑/设定参考，**不可借句式片段**。
   - grep 参考池标准写法：`R="Project/Reference/终局委员会.txt Project/Reference/玩乐关系.txt Project/Reference/术师手册_utf8.txt"`（不含一世之尊、不含文风参考里的仙侠样本）。
8. **片段 grep 命中 ≠ 语用正确（2026-06 Ch18 终轮）**：「钉在」grep 有命中，但参考语境是"皮被钉在木框"的**字面钉**，拿来写"坐着不动"=语用错位。每个片段除了"存在"还要看**它在母句里的真实用法/语域**是否匹配。
9. **过度压缩=电报体=用户"完全没感觉"（2026-06 Ch18 终轮·成功收尾经验）**：片段拼装做成极限压缩会产出冷硬电报体。**轻小说流速=长句呼吸+列举铺排（"有…的，有…的，也有…的"）+夸张怀疑（"几乎要怀疑自己跟踪的是一件行李"）+吐槽织进叙述**。母句范例：「安静得如同一尊雕塑」（术师）、「在站台上重新开始尾随，以免跟丢」（玩乐·逐字级）。最终获用户"好了很多！非常好"确认的 Ch18 版本=这个配方。
10. **"像主角"=场景+对话+世界交互，不是更多内心和背景（2026-06 Ch18）**：背景档案段（"这份工作是三天前接的…三年来都是这样"）是"发给死人的简历"；解释性顿悟（"约纳斯终于意识到…"）也是配角写法。威胁/背景压缩成**一句**、钉在**决断点**上（巷口想起"病院随时可以空出一张床"→咬牙进去），临终念头（那张床/明天的药）就有了着落。给角色至少一次开口说话的世界交互（卖报小贩："看过了。"）。

## A. 三段法流程

### A.1 第一段：节奏分析（写之前）

每段写作前必须明确以下 4 件事：

1. **场景类型**：日常对话 / 战斗紧张 / 感官冲击 / 内心独白 / 探索观察 / 高密度信息 / 黑色幽默 / 余韵转场 / 等等
2. **节奏定调**：
   - **中长句连缀**（默认基础——听日/竹宫风格）
   - **戏剧短独段间插**（少量，仅 1-2 处揭示节点）
   - **对话密集**（对话占主导，叙事极少）
   - **挣扎结构**（每拍"想要 X 但 Y 所以 Z"——见 [[feedback_action_rhythm]] §B）
3. **威胁级别**（如果是危险/战斗）：L0-L4，决定身体反应强度（见 [[feedback_scene_consistency]] §B）
4. **参考锚点**：明确这段对应**哪个参考作品的哪个场景**——后续骨架/填词都从这个参考找

**输出**：一份"节奏蓝图"，500 字以内，给用户确认。

### A.2 第二段：单句句式搭建（在段落骨架确定之后）

**前置：必须先完成第〇铁律的「段落骨架」。** 单句句式是段落骨架内部的填充，不能脱离段落骨架单独做——否则就是"单句都对、整段是 AI"。

**核心铁律：每一句话的句式骨架必须能在参考里 grep 到 ≥1 完全相同结构的句子**。

**操作流程：**

1. **列每一句的骨架结构**，例如：
   - "X 在 Y 的时候，Z 也 W"
   - "X 想 Y，但 Z"
   - "X 没 Y，先 Z"
   - "X 一边 Y 一边 Z"

2. **对每一个骨架 grep 参考**：
   ```bash
   REF="/root/Pi/unnamed/Project/Reference/术师手册_utf8.txt \
        /root/Pi/unnamed/Project/Reference/一世之尊_utf8.txt \
        /root/Pi/unnamed/Project/Reference/文风参考.md"
   grep -h "X.{0,3}Y.{0,5}时候.{0,3}Z" $REF | head -5
   ```

3. **0 命中 = 骨架自造，必须替换**——找参考里真实存在的相似骨架替换

4. **输出**：每句话的骨架+grep 命中证据，给用户确认

### A.3 第三段：填词写作（按骨架填）

**核心铁律：每个不寻常的动词/形容词/名词都必须 grep 验证**。

**操作流程：**

1. **按骨架填词**
2. **每个不寻常词**（不是"的/是/在/他/她"这种功能词）都 grep：
   ```bash
   grep -h "<词>" $REF 2>/dev/null | head -3
   ```
3. **0 命中 = 自造词，必须替换**——找参考里真实存在的同义词
4. **写完一段 ≤150 字立刻给用户看**——不写完整段，控制错误密度

## B. 节奏类型对照表（参考锚点）

| 场景类型 | 参考锚点 | 节奏特征 |
|---|---|---|
| 日常对话+轻小说感 | 龙与虎 / 春物 / 异兽迷城（高阳和妹妹/青灵） | 中长句基础+大量内心吐槽+对话密集 |
| 战斗紧张感 | 异兽迷城（高阳被攻击）+ 术师手册（亚修挨打） | 长句串多个动作+"迫不得已"挣扎词+具体身体感受 |
| 感官冲击 | 异兽迷城（兽化目击）+ 诡秘之主初醒 | 多感官同时（视觉+触觉+听觉）+ 中长句连缀 |
| 探索观察 | 默读（犯罪现场）+ 诡秘之主教堂 | 中长句+环境细节+角色不解释的观察 |
| 内心独白（人性化） | 春物（比企谷自言自语）+ 异兽迷城（高阳吐槽） | 思绪流（一个念头接一个念头）+ "...啊"/"...吧" 口语词 |
| 高压战斗的内心 | 术师手册（亚修被打那段） | 短而准（不啰嗦）+ 真实的痛感和判断混合 |
| 牺牲死亡（克制式） | 诡秘之主奥黛丽告别 | 长句铺垫具体动作+ 关键独段一句话 |
| 黑色幽默 | 春物 / 龙与虎 | 反差感（严肃场景+口语吐槽）+ 不解释 |

**写之前先选定参考锚点**，整段都对照这个锚点写。

## C. 骨架 grep 验证示例

### C.1 实战案例：身体反应骨架

**场景**：阿尔温踏入致死压力的雾里，身体冻结。

**想写的骨架**：
> "X 刚 Y，Z 就 W"——例如 "第一步刚踩到雾里，他就不能动了"

**grep 验证**：
```bash
grep -h "刚.{0,3}，.{0,8}就.{0,6}" $REF | head -10
```

**期望输出**：参考里有大量类似骨架（如术师手册"亚修刚抬头，光就照下来"）—— 命中 = 可用

**反例**：如果我自造一个骨架 "X 的 Y 像被 Z 攥了 W"——这是"无形的手攥住"的变体——grep 0 命中 = 必须替换

### C.2 实战案例：动作序列骨架

**想写**："他想退，但来不及——被踹倒+飞回+摔在地上"

**用户已发布正文 2994 完整对应**：
> "下一秒三楼倒塌的气浪从前面把他整个抛飞出去，胸口磕在身后一根断了一半的钢筋上。肺里的气一下子被碾空。"

**骨架**：动作 1，动作 2，动作 3。最后一个独段"X 的 Y 一下子被 Z"。

—— 100% 贴合用户已建立语感。直接用。

## D. 自检命令（骨架/词级双层）

### D.1 骨架级自检

```bash
# 写完段落后，对每个不寻常的句式骨架问"参考里有吗"
# 例如想验证 "X 的 Y 像被 Z" 这种骨架：
grep -h "的.{0,6}像被" $REF | head -5
# 0 命中 = 该句骨架重写
```

### D.2 词级自检

```bash
# 列出本段所有不寻常的动词/形容词/名词
# 对每一个跑：
for w in "蛛印" "亮的位置" "本能选了"; do
    c=$(grep -h "$w" $REF | wc -l)
    [ "$c" = "0" ] && echo "▲ 自造词: $w"
done
```

## E. How to apply

1. **接到写作任务**后立刻执行 §A.1 节奏分析，输出节奏蓝图给用户
2. 节奏蓝图确认后，执行 §A.2 骨架搭建+grep 验证
3. 骨架确认后，执行 §A.3 填词写作（每 ≤150 字给用户看）
4. **每一步用户都可以打断**——三段法的核心目的就是把错误密度降到最低
