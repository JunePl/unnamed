用户在2026-04-30设定了98%贴合度铁律："每一句话和单词都是能在参考和调研以及两本书中能找到的组合"。

**Why:** AI默认会写"看起来精致但不像人写的"句子。98%贴合度的本质是强迫每一句都过一道"原文存在过吗"的检查，把AI的诗化倾向卡死。**判定标准是"看着像人类在写的小说"，不是"看起来文笔好"。**

## 写作流程（顺序不可颠倒）

### 第0步 - 前置准备（写之前必须做）

1. 确定本段场景类型（战斗/牺牲/逃亡/对话/感官/视角切换等）
2. 读对应的 `Project/Reference/调研_*.md` 原文片段
3. 读 `Project/Reference/文风参考.md` 对应的作者样本
4. 列出本段每一拍的"想要X但Y所以Z"骨架（参考`feedback_action_struggle_pattern.md`）
5. 对应已知场景的语感锚点（见下方"对照原文的来源"）

### 第1步 - 先写200字核心样本

- **绝不直接写完整段**
- 先写200字核心样本（选最考验语感的那段，通常是高强度场景）
- 提交给用户确认大方向

### 第2步 - 提交样本时必须附带

- **逐句对照表**：我的句子 / 对照原文 / 应用了什么技法
- **自检报告**：破折号次数 / "不是X是Y"次数 / "X的那种"次数 / 文艺比喻 / 拟人化怪物 / 现代词污染等
- 不能光给样本结果，必须让用户看到我的对照思路

### 第3步 - 收到反馈后判断错误等级

| 用户反馈 | 错误等级 | 处理方式 |
|---|---|---|
| "差的很远" / "70%" | 大方向错 | 不改细节，先重新检查视角/节奏/挣扎结构 |
| "特别AI" / "这个不行" | 句式拐杖错 | 找出AI拐杖类型，全文扫描 |
| "这次可以，但是注意X" | 大方向对了，细节错 | 修改X，保留其他 |

### 第4步 - 写完整段时

- 每一句话写完都问："这句能在参考里找到对应模板吗？"
- 提交时附带：自检报告 + 后续替换说明（如"X行的米尔顿改成琳莎"）

## How to apply

- 任何"修改/重写正文"任务必走这个流程，不能跳过
- 用户说"差的很远"先想视角错位，不是细节错
- 用户说"特别AI"先想哪个句式拐杖（"X的那种"/"——"/"不是X是Y"/文艺比喻），全文扫
- 提交样本必带逐句对照表+自检报告，让用户判断方向是否对
- 与所有句式禁用类feedback协同（`feedback_no_label_judgment.md`/`feedback_no_negation_pattern.md`/`feedback_emdash_overuse.md`/`feedback_no_literary_metaphor_in_voice.md`）
