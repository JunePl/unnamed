# 伏笔克制铁律

**Why（用户原话）：**
"而且这个伏笔也太明显了。"

我写："蛛印亮起来了，是他二阶那天印上去的那个。**亮的位置正对着中间那尊神像**。" + "蛛印是他二阶那天印上去的那个"

—— **双重明示**+**叙述者直接挂钩两个事物**=废伏笔。读者第一遍就懂了"哦原来这两个有关系"，反转就没了。

---

## A. 伏笔的三个层次（精度从低到高）

### Level 1（最差，禁用）：叙述者直接明示
> 蛛印亮起来了。**亮的位置正对着身后这尊神像**。
> ——叙述者告诉读者"看，这俩有关系"。读者立刻懂。零悬念。

### Level 2（中等，允许但克制使用）：角色注意到关联
> 蛛印亮起来了。阿尔温下意识看了一眼神像。**他没想，但他看了**。
> ——通过角色的不自觉动作暗示关联。读者可能注意到，但不一定。

### Level 3（最好，目标级别）：纯客观陈述+留给读者
> 蛛印整个亮起来了。亮度从指节往手腕退了一波。
> （什么都不挂钩。读者后面知道蛛印=蛛女神时，回头才意识到"哦，刚才印记亮就是因为靠近了神像"）

**目标：90% 用 Level 3，10% 用 Level 2，0% 用 Level 1**。

---

## B. 双重明示禁令

**错（双重明示）：**
> 蛛印亮起来了，是他二阶那天印上去的那个。亮的位置正对着中间那尊神像。

—— 这里明示了两遍：
1. "蛛印是他二阶印的"（提醒读者蛛印的身份）
2. "亮的位置正对着神像"（直接挂钩印记和神像）

**对（单层暗示）：**
> 蛛印亮了起来。

—— 蛛印的身份前文已建立（3471 行已经描述过白蜘蛛真灵），不需要在这里再提醒。"亮了起来"是事实，读者自己去想为什么。

---

## C. 三条铁律

### C.1 禁止"X 是 Y" 的内嵌解释

任何"X，是 Y 的那个/是他在 Z 干的那个" 句式都是 AI 在替读者补课。**已建立的设定不重复**——见 [[feedback_98_percent_process]] §D。

### C.2 禁止用方位/角度明示关联

"X 亮的位置正对着 Y"、"X 的方向就是 Y"、"X 指向 Y" —— 这种"几何明示"都是叙述者在做媒。**让读者用脑子算**。

### C.3 物理动作 ≠ 关联宣示

可以写"蛛印亮了"，可以写"阿尔温看了一眼神像"——但**不能在一个句子里把两个事件用"对着/正对/指向"挂钩**。

---

## D. 修改对照表

| 错（明示） | 对（克制） |
|---|---|
| 蛛印亮起来，亮的位置正对着神像 | 蛛印整个亮了起来 |
| 他低头一看蛛印，再抬头一看神像——形状一模一样 | 他低头一看蛛印。\n\n然后他抬头看了一眼神像。\n\n他没说话。 |
| 小女孩的发色和蛛印的颜色一模一样 | 小女孩浑身雪白 |
| 钟楼里的钟声和当年那一晚一样 | 远处的钟敲了一下 |

---

## E. 自检命令

```bash
# 几何明示词
grep -nE "正对着|正好对着|的方向就是|指向.{0,4}(神像|那)" /tmp/check.txt && echo "▲ 几何明示伏笔"

# X 是 Y 的内嵌解释（已建立设定不重复）
grep -nE "(是他|是那个).{0,8}(那天|那次|那个).{0,8}的" /tmp/check.txt && echo "▲ 重复已建立设定"
```

---

## F. How to apply

1. **写两个事物在同一句出现时**：检查是否在暗示关联
2. **如果在暗示关联**：拆成两个独立句，删掉"对着/指向"等几何词
3. **如果是已建立的设定**：不要重复提醒读者
4. **想做关联的话**：让角色**沉默地看一眼**，不解释；让读者自己看出来
