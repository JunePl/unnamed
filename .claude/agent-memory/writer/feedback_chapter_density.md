# 章节信息密度

**Why（用户原话）：**
"目前的字数在12k有点多了... 现在有的地方偏流水账，很多不需要的信息，很多阿尔温的思考根本不需要现在就写出来，我觉得8千到1万字差不多。"

---

## A. 单章字数标尺

- **目标**：8-10k 中文字 / 章
- **超过 12k 字 = 信息密度太低**，必须砍
- **少于 6k 字 = 信息密度太高**，可能省略了必要的对话/铺垫

按 wc -m 算大约 1.4 倍中文字：
- 11500-14500 字符 = 8-10k 中字（达标）
- > 17000 字符 = 太长

---

## B. 应删的"阿尔温内心总结"句式

以下是高频踩雷模式——**读者第一遍就能看出，叙述者画箭头反而是冗余**：

| 黑名单句式 | 为什么 |
|---|---|
| "X 在心里把这条挂了一格" | 读者知道阿尔温在记线索，不需要叙述者点 |
| "X 在心里把时间线对了一遍。8:20 是 A，8:40 是 B，8:41 是 C..." | 时间线对照让读者自己做。如果剧情后续真用到，到时候再说 |
| "X 在心里把这条挂在了 Y 旁边。两个版本暂时不冲突..." | 矛盾让读者发现，不要叙述者解释 |
| "X 把这条对了一下文件袋里的笔录" | "对了一下"是动作但接的是结论。读者不需要这个对照过程 |
| "时间线对得上，反应也对得上" | 调查总结。每个场景一段总结 = 流水账 |

### B.1 替代方法

让对话本身说话——不再加内心总结句。如果伏笔真的重要，用：
- 角色再次提到（如海伦娜也说"娘家事多"，读者自然对照）
- 物理动作（如阿尔温把笔录折成两折塞回袋）
- **章末或下一章** 才用阿尔温内心点明（不在当场点）

### B.2 例外（允许保留的内心）

- **当时就要决策的**："这种谎不难拆" → 决定下面怎么问
- **角色特征的吐槽**："数字最大那个不一定就是凶手，这是阿尔温在执法局第一年学到的事" → 展示阿尔温的智识背景
- **梓川式松弛吐槽**（feedback_alwin_persona.md）：玩笑式内心，不是分析式

---

## C. 应删的"流水账细节"

- **场景过渡**："X 朝那边走过去" + 下一段开头"X 走到 Y 房间" → 合并成一句
- **物品细节重复描写**：阿尔温的咖啡袋在每个房间都说放在哪儿 → 关键位置说 1-2 次就够
- **配角的"背景对话"**：如"上一位助理走得仓促，我来的时候连客户名单都是散的，花了大半年才理出来" → 删（除非这个背景是伏笔）
- **重复的角色判断**："X 是一个 [形容词] 的人" → 删（让动作说话）
- **过度详细的外貌细节**：6 句视觉细节 → 3 句

---

## D. 应删的"长劝说"

阿尔温的"四句台阶式劝说"（"哈罗德留您不是怀疑您。是因为留在画廊里能让您先睡一觉，比让您回家路上被人撞见好。您要是想告诉我您在打盹，我能写在笔录里。要是您坚持您一直在岗..."）→ 缩到 2 句即可。

替代：
> "罗尔斯先生，"他把声音放低了，"哈罗德留您不是怀疑您。您打盹的话我写笔录，您回家；您坚持一直在岗，我就再翻一遍门禁。"

—— 砍 60-100 字，但选项压力一点不少。

---

## E. How to apply

写完段落或章节后跑一遍：
1. **找所有"X 在心里把...挂了/对了/记下了"句子** → 默认全删
2. **找所有"X 想：长段分析"段落** → 50% 删掉
3. **找所有"X 朝 Y 走"+下段开头重复** → 合并
4. **看字数**——12k 中字以上必须找冗余删
