# 场景一致性铁律

**Why（用户原话）：**
- "磨得很细、很彻底，边缘的灰还没沉下去 啥意思？"——把犯罪现场的描写硬塞给神殿
- "动都不敢动，胃里面还能翻？"——死亡威胁下还能"胃里翻"
- "人脸不敢看却能看到眼睛很多"——逻辑矛盾
- "你手上就一个印记还能决定怎么亮亮哪里？"——物理逻辑漏洞

---

## A. 描写必须匹配场景类型

### A.1 错位案例

| 错（场景错位） | 为什么错 |
|---|---|
| 神殿底座"边缘的灰还没沉下去" | "灰未沉=痕迹新"是**犯罪现场调查**的描写。神殿哪来的灰？为什么会有灰沉/未沉？这是把 A 场景的常用描写硬塞给 B 场景 |
| 异界森林"现场没有挣扎的痕迹" | "挣扎痕迹"是法医语言，异界森林用不上 |
| 王宫宴会"灰扑扑的窗帘" | 王宫窗帘不会"灰扑扑"，除非这是被遗弃的王宫 |

### A.2 修复方法

**步骤 1**：写每段描写前列出"这个场景的真实属性"
- 神殿：石头、灰尘、阴冷、回声、年代久远、被信徒拜访过的痕迹
- 灰烬平原：骨粉、风、灰雾、空旷
- 办公室：纸张、墨水、阳光、地板嘎吱声

**步骤 2**：检查每个描写元素是否在该场景的"真实属性清单"里

**步骤 3**：如果描写元素不在清单里，问"这里为什么会有 X？" 如果回答不上，删除

## B. 反应强度必须匹配威胁级别

### B.1 威胁分级 → 反应强度对照表

| 威胁级别 | 触发感受 | 身体反应 | 错误写法（强度不匹配） |
|---|---|---|---|
| **L0 不适** | 闷热、轻微头痛 | 揉揉头、动一下身体 | （这级别 OK） |
| **L1 警觉** | 有人在看我 | 寒毛立起、慢慢回头 | （这级别 OK） |
| **L2 真实危险** | 这地方不对 | 喉咙发紧、走得慢了 | "胃里翻一下"（OK） |
| **L3 致命压力** | 再走一步就会死 | **身体冻结、不敢动、连呼吸都停了** | "胃里翻一下"（**错！太温和！**） |
| **L4 临死瞬间** | 已经被刺中 | **疼痛淹没所有感知、视野黑/白闪、思维一片空白** | "皱了皱眉"（**错！太克制！**） |

### B.2 这次踩雷案例

**错（致命压力配 L2 反应）：**
> 一种死一样的压力从前面压过来，他突然喘不过气，胃里翻了一下又翻一下，心也跟着被一只无形的手攥住，攥得越来越紧。

—— 用户在说"踏一步就会死"（L3+），但我用了"喘不过气+胃里翻+心被攥"——这是 L2 级别的描写。**强度严重不匹配**。

**对（L3 致命压力的真实反应）：**
> 第一步刚踩进雾里，他就不能动了。

> 不是不敢——是身体不让他动。一种比"害怕"更早的反应：肺里的空气一下子卡在喉咙后面，吸不下去也吐不出来；脖子后面的汗瞬间冷了，皮肤紧得像被什么东西从背后扯住；他的脚不肯往前抬，也不肯往后挪。

> 他听见自己的心跳了——不是变快，是变得很清晰，每一下都像有人在他胸腔里敲一下。

—— **关键**：致命压力下身体首先是**冻结/无法行动**，然后才是各种"清晰得不正常"的感官细节。**不能写"胃里翻"这种小不适**。

### B.3 反应强度自检

写完段落问：
- 这个威胁是什么级别？
- 我写的反应是这个级别该有的吗？
- 如果换成普通的"被人吓一跳"（L1），我的反应描写还成立吗？如果成立 = 我写得不够强

## C. 行为与感知逻辑一致

### C.1 矛盾案例

**错：**
> 神像脸上的部分他不敢再往上看了。眼睛太多。

—— 既然不敢看，**怎么知道眼睛多？怎么数？**

### C.2 修复方法

**选项 A（先看见 → 才不敢看）：**
> 他视线扫到神像的脸，一下子就移开了。眼睛不止两只——是叠在一起的好几对，分上下两排嵌在脸上。再多看一眼他不敢。

—— 先**一瞥看见**，意识到不对，**才不敢继续**。

**选项 B（描述细节再描述移开）：**
> 他抬头扫了一眼那张脸。
> 
> 然后他就低下头了。

—— 不解释"为什么不敢"，让读者自己懂。**留白比解释更有冲击力**。

### C.3 自检

任何"X 不敢/不能 Y" + 紧接"Z 是 W"（描述 Y 应该看不到的东西）= 逻辑矛盾。必须改成"先 Y 一瞥 → 才不敢 Y"或"完全留白"。

## D. 物理逻辑自洽

### D.1 错例

**错：**
> 蛛印亮起来了，是他二阶那天印上去的那个。**亮的位置正对着中间那尊神像**。

—— **一个**印记是整体亮的（或者整体不亮），不能"哪个位置亮"。除非印记是个面积大的图案——但前文 3471 定义白蜘蛛真灵"巴掌大小"，印记不是大图案。

### D.2 修复方法

**步骤 1**：每写一个"奇异现象"问：物理上**真的能这样吗**？

**步骤 2**：如果不能，改成符合物理逻辑的描写。
- 改例："蛛印整个亮起来了，亮度从指节往手腕一波一波退。"（整体亮+变化）
- 改例："蛛印亮起来，亮的方向他不用看也知道——就是身后那尊神像。"（用阿尔温的感知替代物理"指向"）

### D.3 自检

写任何超凡现象/异常感官后，问：
- 这在物理上能成立吗？
- 单个/小物件能"有方向地"做事吗？
- 如果不能，换成"角色感知"而非"物理表现"

## E. 综合自检命令

```bash
# 场景错位高频词（写到对应场景外时检查）
grep -nE "灰还没沉|边缘的灰" /tmp/check.txt && echo "▲ 灰未沉=犯罪现场用语"

# 致命威胁场景下的温和反应词
grep -nE "(死.*压力|致命|会死).{0,30}(胃里翻|皱了皱眉|愣了一下)" /tmp/check.txt && echo "▲ 反应强度不匹配威胁"

# 不敢 X + 接 X 的描述（矛盾）
grep -nE "不敢[一-龥]{1,3}(看|听|碰)" /tmp/check.txt
# （需要人工检查后面是否描述了"不敢"的东西）
```

## F. How to apply

1. **写之前**：明确场景类型 + 威胁级别
2. **写时**：每个描写元素问"这场景该有吗？" + 每个反应问"威胁级别配吗？"
3. **写完**：跑自检命令 + 人工过一遍逻辑一致性
