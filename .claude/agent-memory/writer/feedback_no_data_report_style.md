---
name: 禁止信息汇报式写作
description: AI倾向于"汇报信息"而非"经历事件"——砍掉了过程和质感只剩干巴巴的结果，读起来像游戏日志而不是人在讲故事
type: feedback
---

AI的核心倾向：把"直白/碎片式"理解成"数据报告"，写出来的东西像系统日志而不是活人的体验。

❌ 典型错误：
- "阿尔温等了一轮。" → 游戏回合制语言
- "预见更新：变异体照常" → 系统通知
- "剩一发。" → 弹药UI
- "零伤害。" → 伤害面板
- "他之前对着巨眼轰了五发才打出一条缝，两秒合上" → 干巴巴的结果汇报，没有过程

✅ 正确写法（用户示范）：
- "阿尔温又一次用预见眼躲开了攻击" → 人在做动作
- "之前全神贯注更是靠着单镜所展现的灵能痕迹才做到四颗子弹打在一条线上，但变异体只花了几秒就重新愈合" → 有付出的努力（全神贯注）、有方法（靠着单镜）、有具体成果（四颗子弹打在一条线上）、有对比（但变异体只花了几秒）

**规则：信息必须通过"人经历了什么"来传递，不能直接列举结果。**

- "碎片"是人思绪流里自然断裂的碎片，不是Excel数据条目
- "直白"是用简单的话说清楚一件事的前因后果，不是只说"剩一发"
- 每一个战斗动作都要有：谁做了什么 + 怎么做到的（哪怕一句话带过）+ 效果
- 回忆之前的战斗不能用干数据（"五发打出一条缝"），要带上当时的体验质感（"全神贯注才做到四颗打在一条线上"）

**Why:** 用户发现我在去文艺化的过程中矫枉过正，把"过程和质感"也一起砍掉了，剩下的只是信息骨架。

**How to apply:** 写完每个段落后自检——这段话像是一个人在讲他的经历，还是像在读一份报告？如果像报告，就加回过程和质感。
