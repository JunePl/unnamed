# 内心独白方法论

## A. 密度：内心声音占比 25-35%

**Why:** 文风参考中的作者——竹宫 / 听日 / 不讲理——内心独白密度极高（30-40%）。"心里慌得一批""但是再烦躁也不能拿镜子怎样啊！""好不甘心啊"——这些不是传统意义上的"心理描写"，是**角色的声音**，是叙事的核心质感。压到 10% 等于把角色变成提线木偶。

**核心区分：**

- **角色内心声音**（用大量）：角色用自己的口语在脑子里说话
- **传统心理描写**（仍然禁止）：叙述者用文学语言解说角色心理

**仍然禁止的：** 叙述者视角的情绪形容词堆砌（"他感到无比悲伤"）、替读者感受、用内心独白点明伏笔。

**自检：** 读完一段问——"我知道这个角色此刻在想什么、感受什么吗？"不知道就缺少内心声音。

## B. 人性化（不机械分析）

AI 写内心有系统性毛病，全部来自"精确高效传递信息"的倾向。

### B.1 身体感受用口语反应 + 人格滤镜，不过医学滤镜

| 错 | 对 |
|---|---|
| ❌ "那个位置在左胸偏下，规律地跳了二十四年"（解剖学定位） | ✅ 竹宫写鹦鹉眨眼："令人不快且意义不明的眼皮痉挛"（主观、好笑） |
| ❌ "胸口磕过的那块每跳一次都疼一次" | ✅ 用户正文："胸口磕的那一下让他每呼吸一次都要倒抽一口冷气" |
| | ✅ 不讲理写摔倒："总觉得脑袋有点刺痛"（轻描淡写） |

**规则：** 身体感受过人格滤镜（这人会怎么吐槽这个感觉），不过医学滤镜。

### B.2 让角色自己判断、想错、纠正、得出结论

不要替角色把活干了。

- ❌ "胸口有东西在动。灵质体。"（作者直接告诉读者答案）
- ✅ 用户示范：以为是心跳 → 嫌弃心脏不争气 → 发现位置不对 → "这是……那个灵质体？"

**规则：** 让角色经历"误判 → 纠正 → 理解"的过程。

### B.3 自嘲是角色自然产生的念头，不是对读者说的段子

- ❌ "大概死人没这待遇"（说给读者听的俏皮话）
- ✅ "嫌弃自己心脏太不争气"（阿尔温自然会想的事，他在嫌弃自己）

**规则：** 内心吐槽的对象应该是自己、眼前的事物、认识的人，不是面向读者。

### B.4 闪回让角色"走进场景四处看"，不是闪一张图就走

- ❌ "淡紫色的荧光。地下通道。岔路口。"（静态碎片）
- ✅ 用户示范：先失去意识 → 华尔德的动作 → "阿尔温打量着四周" → 描述环境 → "总感觉这里是哪种恐怖电影里不知死活的主角才会来的地方"

### B.5 转场直接切，不解释"转场是怎么发生的"

- ❌ "然后记忆涌上来了，没有预兆，不是他想的，是被塞的"（四层解释同一件事）
- ✅ 竹宫："搞不好今天能办到！" → 直接下一行 "波！"
- ✅ 不讲理："那一天。我脑里是这种想法。★★★ 之后过了一个月。"

### B.6 世界观信息通过"角色已有知识 vs 眼前现实"的落差嵌入

- ✅ "奥里斯塔尼有迹可循的历史中，关于灵质体的记录少得可怜，但绝对不包括融入身体和狂跳这点"
  - 一句话完成：世界设定补充 + 幽默落差（"手册上没写这个功能"的感觉）

## C. 不写"信息汇报式"（游戏日志感）

AI 把"直白 / 碎片式"理解成"数据报告"——只剩干巴巴的结果，没有过程和质感。

### C.1 典型错误（游戏日志感）

- "阿尔温等了一轮。"（游戏回合制语言）
- "预见更新：变异体照常"（系统通知）
- "剩一发。"（弹药 UI）
- "零伤害。"（伤害面板）
- "他之前对着巨眼轰了五发才打出一条缝，两秒合上"（干巴巴的结果汇报，没有过程）

### C.2 正确写法

- ✅ "阿尔温又一次用预见眼躲开了攻击"（人在做动作）
- ✅ "之前全神贯注更是靠着单镜所展现的灵能痕迹才做到四颗子弹打在一条线上，但变异体只花了几秒就重新愈合"

  关键差别：有付出的努力（全神贯注）+ 有方法（靠着单镜）+ 有具体成果（四颗子弹打在一条线上）+ 有对比（但变异体只花了几秒）

### C.3 规则

- 信息必须通过"人经历了什么"来传递，不能直接列举结果
- "碎片"是人思绪流里自然断裂的碎片，不是 Excel 数据条目
- "直白"是用简单的话说清楚一件事的前因后果，不是只说"剩一发"
- 每一个战斗动作都要有：谁做了什么 + 怎么做到的（哪怕一句话带过）+ 效果
- 回忆战斗不能用干数据，要带当时的体验质感

**自检：** 这段话像是一个人在讲他的经历，还是像在读一份报告？

## D. 作者情绪 vs 角色情绪分离（最高优先级）

**这一节优先级高于场景工具箱里任何技法。所有场景写作前必须先过这一关。**

### D.1 铁律

写作时必须分离两种情绪来源：

- **作者情绪**（AI 觉得这件事很酷/很震撼）——只存在于写作视角，**不能进入文本**
- **角色情绪**（角色在这个情境下真的会反应什么）——这才是文本里应该出现的

**绝对禁止**把作者情绪硬套到角色身上。典型错误：

- AI 觉得"八千年前的人也喝茶"很震撼 → 让角色"脊背发凉" → **错**。古人喝茶不值得震惊。
- AI 觉得"镜面反射异界景象"很美很神秘 → 让角色"呼吸停了一下" → **错**。彼界本来就什么都可能发生。
- AI 觉得"真相揭露的瞬间"很酷 → 让角色"瞳孔一缩心跳漏了一拍" → **错**。侦探见过场面。

### D.2 元规则：事件稀有度决定情绪强度

> **这个事件在这个世界观里，对这个角色而言，有多稀有？稀有度决定角色反应强度。**

- 彼界里看到上个纪元影像 → 彼界本来就什么都有 → **稀有度低 → 调侃一句就过**
- 影像里出现一个他认识的人 → **稀有度中 → 角色真的停一下**
- 影像里出现至亲的脸 → **稀有度拉满 → 角色会真的动**

**情绪峰是稀缺资源，不是每段都该用。**《默读》《春物》《异兽迷城》的好场景大量是平淡处理——因为事件不配强情绪。AI 训练数据偏好"高情绪密度"是必须主动对抗的偏差。

### D.3 五道强制检查（每段写完都要过）

#### 检查 1：角色反应卡（写之前）

动笔前先在脑子里填一张卡：

- 这段发生了什么事件？
- **以具体角色的性格**，他第一反应是什么？
- 反应强度定级：**0**（无动于衷）/ **1**（调侃一句）/ **2**（警觉）/ **3**（认真分析）/ **4**（真的震动）
- 绝对不用哪些反应？（例如阿尔温几乎不会"脊背发凉""后背发凉""呼吸停了一下"）

#### 检查 2：情绪匹配度（写之后）

扫描所有情绪词（震惊 / 愣住 / 脊背发凉 / 心跳加快 / 呼吸停了 / 握紧拳头 / 后背发凉 / 瞳孔一缩 / 喉咙发紧……），逐个问：

- **事件配得上这个情绪吗？**
- **这个角色的性格会产生这个情绪吗？**
- 不配就降级（震惊 → "哟，有意思"）或删除。

#### 检查 3：碎句意义检查

每个独立短句问：**读者读完能用一句话回答"这句在说什么"吗？**

- 能答 → 保留
- 不能答 → 改写或删除
- 特别警惕：三个名词堆叠+省略号、"没认出来的XX"、"他没问/没说话/想了想"偷懒拐杖、单句独占一行的陈述句

#### 检查 4：作者视角 vs 角色视角分离

每段写完问：**这段的情绪，是"我（AI）觉得酷"还是"角色觉得值得反应"？**

- 前者 → 删掉作者情绪
- 后者 → 保留

#### 检查 5：AI 偏好反向扫描

- 堆强情绪词了吗？
- 用三名词碎句装意境了吗？
- 让角色对"AI 觉得酷的事"做过度反应了吗？
- 把"看起来很文学"当成好句子了吗？

### D.4 情绪词黑名单（日常场景慎用）

以下词用在非"稀有度 4 级"事件上，99% 是作者情绪硬套：

- 脊背发凉 / 后背发凉
- 呼吸停了一下 / 屏住呼吸
- 瞳孔一缩
- 心跳漏了一拍 / 心跳加快
- 愣在原地
- 喉咙发紧
