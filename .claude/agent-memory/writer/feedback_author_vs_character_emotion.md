## 核心规则

**写作时必须分离两种情绪来源：**
- **作者情绪**（AI 觉得这件事很酷/很震撼）——这是写作视角
- **角色情绪**（阿尔温在这个情境下真的会反应什么）——这是文本内应该出现的

**绝对禁止**把作者情绪硬套到角色身上。典型错误：
- AI 觉得"八千年前的人也喝茶"很震撼 → 让阿尔温"脊背发凉" → 错。因为"古人喝茶"不值得震惊，阿尔温的性格也不会震惊。
- AI 觉得"镜面反射的异界景象"很美很神秘 → 让阿尔温"呼吸停了一下" → 错。彼界本来就什么都可能发生，不稀奇。

## 元规则：事件稀有度决定情绪强度

> **这个事件在这个世界观里有多稀有？稀有度决定阿尔温的反应强度。**

- 彼界里看到上个纪元的影像 → 彼界本来就什么都有，**稀有度低 → 调侃一句就过**
- 影像里出现一个他认识的人（本不该在这里）→ **稀有度中 → 阿尔温真的停一下**
- 影像里出现米娅的脸 → **稀有度拉满 → 阿尔温会真的动**

**情绪峰是稀缺资源，不是每段都该用。**《默读》《春物》《异兽迷城》的好场景大量是平淡处理——因为事件不配强情绪。

## 五道检查（每段写完过一遍）

### 检查 1：角色反应卡（写之前）
动笔前在脑子里填：
- 这段发生了什么事件？
- **以阿尔温（不是"主角"、不是"我"）的性格**，他第一反应是什么？
- 反应强度：0（无动于衷）/ 1（调侃一句）/ 2（警觉）/ 3（认真分析）/ 4（真的震动）
- 绝对不用哪些反应？（阿尔温几乎不会"脊背发凉""后背发凉""呼吸停了一下"）

### 检查 2：情绪匹配度（写之后）
扫描所有情绪词（震惊/愣住/脊背发凉/心跳加快/呼吸停了/握紧拳头/后背发凉/瞳孔一缩……），逐个问：
- **事件配得上这个情绪吗？**
- **阿尔温的性格会产生这个情绪吗？**
- 不配就降级（震惊 → "哟，有意思"）或删除。

### 检查 3：碎句意义检查
每个独立短句问：**读者读完能用一句话回答"这句在说什么"吗？**
- 能答 → 保留
- 不能答 → 改写或删除
- 特别警惕：三个名词堆叠+省略号 / "没认出来的XX" / "他没问/没说话/想了想" / 单句独占一行的陈述句

### 检查 4：作者视角 vs 角色视角分离
每段写完问：**这段的情绪，是"我（AI）觉得酷"还是"阿尔温觉得值得反应"？**
如果是前者 → 删掉作者情绪，用阿尔温的实际反应替换（或无反应）。

### 检查 5：AI 偏好反向扫描
针对 AI 已知偏好，反向查：
- 堆强情绪词了吗？
- 用三名词碎句装意境了吗？
- 让角色对"AI 觉得酷的事"做过度反应了吗？
- 把"看起来很文学"当成好句子了吗？

## Why:

用户在 Ch16b Part 1+2 重写时发现系统性问题——我（AI）一直在"替角色感受"：我自己觉得"八千年前的人在喝茶"很震撼，于是让阿尔温"脊背发凉"。但这是作者情绪硬套角色身上。阿尔温作为松弛嘴贱型侦探，见过场面，他在彼界看到上个纪元影像时根本不会震惊——彼界本来就什么都可能发生，稀有度低。用户原话："就算有震惊也应该是很特别的事""阿尔温的性格就不会这样"。

更根本的问题：AI 训练数据偏好"高情绪密度"——每个"有信息量的瞬间"都配情绪峰，每个"重要场景"都堆强情绪词。但真正好的小说情绪峰是稀缺资源。

## How to apply:

1. **写每段之前**——先填"角色反应卡"，明确角色反应强度 0-4。不能跳过。
2. **写每段之后**——过五道检查，特别是检查 2（情绪匹配度）和检查 4（作者 vs 角色视角分离）。
3. **情绪词黑名单**（阿尔温场景慎用）：脊背发凉、后背发凉、呼吸停了一下、瞳孔一缩、心跳漏了一拍、愣在原地、喉咙发紧。这些用在阿尔温身上 99% 是作者情绪硬套。
4. **碎句意义硬指标**：写完扫所有独立短句，每句必须能用一句话回答"这句在说什么"。不能回答的删改。
5. **事件稀有度自问**：每次给角色配情绪前问——"这件事在这个世界观里，对这个角色来说，真的稀奇吗？"不稀奇就降级到 0-1 级反应。
6. 与已有 memory `feedback_alwin_sakuta.md` / `feedback_no_literary_metaphor_in_voice.md` / `feedback_no_mechanical_inner_voice.md` 协同使用——这条管"情绪强度定位"，其他管"情绪表达形式"。
