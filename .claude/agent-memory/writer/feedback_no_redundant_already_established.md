# 前文已建立的事物不要重复描写——冗余=AI拐杖

## 规则

新段（重写段/续写段）的描写焦点必须放在**新信息**：
- 状态变化（前文是 A，现在变成 B）
- 新事件（前文没发生过的事）
- 张力变化（前文是稳定，现在是失控）
- 新角色反应（之前没出现过的内心/行动）

**禁止重复描写前文已经讲清楚的：**
- 角色当前位置（除非位置变了）
- 角色当前持有/使用的物品（除非物品状态变了）
- 角色能力/真灵的机制描述
- 已经写过的动作姿势

## Why

AI 默认行为是"怕读者忘了，每段都重新介绍一次"。这导致：
- 每段都在重复同样的视觉锚点
- 读者疲劳（读到第三遍"右手掌心朝着变异体"会出戏）
- 文字密度膨胀但信息密度不变
- 节奏失去推进感

具体踩雷案例（Ch3244 重写）：
- 前文 3216 行已经写：「哈罗德的左手攥着一截还在烧的灰白色火苗...右手掌心朝着变异体，代表司契的印记悬在他掌心上方」
- 前文 3224 行已经写：反噬+流血+寿命燃烧
- 我在重写段还写："悬在掌心的「司契」契书没掉" — 完全冗余，前文已建立

用户原话："悬在掌心的「司契 」契书没掉？？太AI了"

## How to apply

写重写段前：

1. **读前文锚点段**（重写段前 30-50 行），列出前文已经建立的：
   - 物理位置
   - 角色姿势/手部状态
   - 持有物品
   - 能力机制
   - 视觉特征

2. **重写段只写"差量"**：
   - 反噬加剧 → 写"抖得越来越厉害"（变化）
   - 但维持状态没变 → **不要再写**"印记还悬着""契书没掉"
   - 巨眼变弱 → 写新的视觉信号（瞳孔发散、控物乱）
   - 黑泥转向 → 写新的运动方向

3. **每写一句问自己**：
   - 这句话传达的信息，前文是不是已经有了？
   - 如果是，删掉。
   - 如果是部分新（A 维持但 B 加剧），只写加剧的部分。

## 反面例子 vs 正面例子

**反面（AI 默认）：**
> 哈罗德的左手在抖。是反噬，指尖、手腕、肘弯都在抖个不停。他的右手却纹丝不动，掌心朝着变异体，悬在掌心的「司契」契书没掉。

—— "掌心朝着变异体" 和 "悬在掌心的契书没掉" 都是前文 3216 已建立的，是冗余。

**正面（写差量）：**
> 哈罗德咬着牙站在烧烤店的废墟堆顶上。他的左手抖得越来越厉害，掌中烧到大半的契书火光忽明忽暗，鼻孔、嘴角、耳道里的血没停过。他的右手却一动不动。

—— "抖得越来越厉害" 是变化，"烧到大半" 是契书燃烧进度的变化，"血没停过" 是反噬持续。"右手却一动不动" 是和前文对比的张力（左手越抖越厉害 vs 右手稳）。每一句都在写差量。

## 验证方法

写完一段，逐句问：
- "这一句信息在前文是不是已经讲过？"
- "如果是，我有没有写出新的'变化'，还是只在重复'同样的状态'？"
- "删掉这一句，读者会不会失去信息？" 如果不会 → 删

任何一句答"重复且没变化"，必须删或改写为"差量描写"。
