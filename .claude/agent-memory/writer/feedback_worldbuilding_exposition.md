世界观信息的嵌入不是越克制越好。关键原则：**不能让读者一头雾水。**

**正确方法**：从视角角色（主角）的周围体现——
- 主角的内心想法（"如果带着单镜，可能已经开始警告我异常灵能反应了"）
- 对话中自然带出（"沃伦那边的实验室有不少新药，不需要灵晶不需要术法"）
- 叙述中简洁定义（阿克曼症开头的一句话定义）
- 主角的观察推断（"多半是家里在给灵晶矿工厂打工"）

**可以慢慢来，但必须让读者跟得上。**

**Why:** 用户Ch7改稿中在开头直接加了阿克曼症的定义段，给灵晶病加了视觉描述和背景推断。我的draft过度克制，读者可能对"这个病到底是什么"一头雾水。

**How to apply:**
- 新概念第一次出场时，用1-2句话从主角视角给出基本定义，不需要绕弯子
- 之后的细节可以通过对话、行为慢慢补充
- 判断标准：如果一个普通读者读到这里会问"这是什么"，那就需要解释
- 解释的载体优先级：主角内心观察 > 角色间对话 > 叙述直接说明
- 禁止百科词条式展开，但一句功能性定义是必要的
