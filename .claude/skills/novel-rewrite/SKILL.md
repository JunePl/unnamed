# 正文写作 / 重写工作流程（递归骨架法）

**触发条件：** 用户要求修改、重写、新写、续写 `/root/Pi/unnamed/正文/` 目录下的任何小说正文。

**核心铁律（唯一）：递归骨架填空法**——成稿的每一句、每一短语、每一词，都从九本参考里 grep 到真实母句"填空换专名"而来，**没有任何一处是自己组织/拼接出来的**。做对这一步 AI 味自然消失；做错，文笔再好也是 AI。**不能 98% 贴合就不写。**

**⛔ 只准 grep 这九本参考找母句（八本他人作品＋你自己的《世界末日的世界录》，都不是本作 unnamed）。禁止 grep、参考、借用本作正文前面的章节。** 本作前文是草稿、本身可能带 AI 味，拿它当语感/句式参考＝复制污染。剧情逻辑、术语、设定的继承，靠读者视角档案（`Project/读者视角/`）和细纲，**不靠 grep 正文**。正文上下文只在"确认剧情怎么接得上"时读一眼，绝不抄它的句子、绝不拿它当语感锚。

## 三条不可绕过的硬验证

### 硬验证 1：每次提交样本必带"已读参考清单"
```
【已读参考】
- 调研文件：[路径] —— 这次读到的有用句式/语感
- 九本 grep：[搜了什么关键词] —— 找到哪些可整段/整句套的母句，给【文件:行号】
```
没有这段清单 = 没读 = 不能交稿。

### 硬验证 2：每次提交必带"grep -Hn 自检日志原始输出"
不是"我跑了，全 0"的声明——是终端实际输出。

### 硬验证 3：用户负反馈"差的很远 / 不对"——禁止局部修补
- 第一次：停下，重新检查"母段选得对不对、视角对不对"，向用户汇报。
- 第二次：禁止再写样本。停下让用户口头解释他要什么，**用文字复述给他确认**，确认后再写。

## Step 2 — ⭐⭐⭐ 递归骨架（整个 skill 唯一核心，写作成败全在这一步）

**这一步做对，AI 味自然消失；做错，后面 verify 只是贴皮。**

### 2.0 核心定义

**成稿每一句 / 每一短语 / 每一词，都从九本 grep 到真实母句"填空换专名"而来，无一自创。** 骨架是递归三层：

- **大骨架 = 一整段「连续的、来自同一处」的真实母段。** 整段搬过来，只换专名、让剧情迁就它的结构，**一个字都不自己造**。它决定这段由哪些小句、按什么情绪推进与顺序构成。
  - ❌ **东拼西凑贴标签（头号错误）**：从多个不同出处各捡一个短语拼成一段——每句怎么连全是自己写的，这不是骨架，是贴皮。
  - ❌ **先列叙事拍点 / 节奏蓝图再找词填**：一旦先写"这段写什么、怎么推进"，句子结构就已自创。**跳过拍点，唯一动作＝先锁定一整段连续母段、让它决定写什么。**
  - ✅ **唯一合格形态**：找到同一处的一整段连续母段或者整个段落的完整母句骨架，整段逐句套，给用户看"母段原文 vs 成稿"逐句对照。
  - 场景在九本里找不到同剧情的母段时，找**修辞/情绪类型同构**的连续母段（杀手死巷→"被逼绝境/猝死/临终念头"类的母段；上位者办公室谈话→"上位者通报局势/交代安排"类的母段）。同构指情绪与修辞结构同构，不是剧情场景同构。

- **小骨架 = 每个小句各自的一条完整句式母句。** ⚠️**最易做错的一层**：小骨架是"一整句的句式母句"，不是几个词。grep 到"弯腰""捡起""青石板"再拼成「弯腰从青石板上捡起手环」＝错（句式自创）；必须找到「(薇瑟)从地上捡起狐狸假面」这样**一整句**，套成「从地上捡起手环」。**找词 ≠ 找小骨架。**

- **词 = 小骨架空位里换进去的专名**，也要各自 grep 到确切组合。

### 2.1 🟥 S 级三铁律（一条都不能破）

1. **一句一母句，不准缝合两母句。** 母句「回身架起步枪」完整了，不许接「扣下扳机」凑成一句；要的动作链得找本身就完整包含它的母句，否则拆成多个独立小句、各对一条母句（由大骨架定顺序，不算缝合）。
2. **不准用孤立词拼出没有母句结构的句子。** 「落了空」+「空无一人」拼一句、「左右两边各站着一个」＝句子结构没母句，禁止。**叙述句容易盯紧，动作/画面流水（弯腰捡起、买面包、群像构图、人物登场）最易松懈退化成词拼——盯死。** 群像找「穿着体育服站成一排」这种真实群像母句。
3. **每个确切组合 grep -Hn 到，近义/同词根不算。** 「没停」不能拿「没停下」当出处；「那一枪」三本皆无就弃用。换进去的专名也要各自 grep 到确切组合。

### 2.2 🟥🟥🟥 整句套用铁律（最根本）

**成稿 = 一条完整母句，原样保留它的全部结构、长度、标点、所有非专名的字，只换里面的专有名词。** 三种"借壳"全禁：
- **不许截取母句的一部分**：「她融入人群里继续寻找虚实素材」不能只挖"融入人群"丢掉后半自己接。
- **不许加减/改动标点、改结构**：「扶着剑袋大口喘气」不能改成「扶着墙，大口喘气」。
- **不许把词组拆出来重组成新句**：「变到再也回不来的地方」不能拆出"再也回不来"重组成「人要是没了就再也回不来了」。
- **不许为塞剧情强改母句的词**：「静静地放下手机」就写"放下手环"，别改"记下"。

**判定（用户原话）：你想写的意思，参考里找不到一条结构/长度都吻合、只需换专名的完整整句 → 说明这写法太烂、没人会这么写，放弃它，改用参考里真实有整句的表达。** 不是去截取/改标点/拆词把它硬凑出来。

**✅ 别教条到瘫痪：** 保留母句完整句式结构/标点/节奏的前提下，可以——换专名（人/物/地）、换人称代词、为通顺做近义动词微调。范例：术师15690「剑姬魔女**看见**石头在**观者**手里忽然化成齑粉，顺着他的指间滑落下来」→「雷纳德**看着**手环在**自己**手里忽然化成齑粉，顺着他的指间滑落下来」（换主语+近义动词，结构一字未动＝合格）。判别线：结构完整保留=合格；截半句/改标点节奏/拆词重组=破坏。

### 2.3 操作流程（每段都走一遍）

1. **定效果**：这段要达到什么（紧张/吐槽/落差/悲凉/神秘/克制/上下级博弈…）。
2. **找大骨架**：去九本找一个达到相同效果的真实连续母段，read 出来。没有大骨架 = 不能写。
3. **逐小句找小骨架**：大骨架每个小句位置，去九本找一条整句结构都能套的真实母句。只换专名，其余句式/动词/连接词/副词一字不动。
4. **递归下钻**：某个小骨架不足以表达 → 把它当新骨架，内部继续找小小骨架，直到触底为词。
5. **逐组合 grep -Hn 复核**：成稿每个 2-4 字短语回 grep（带文件名），确认确切组合存在、且语域匹配。任一处 grep 不到 → 改写迁就母句，绝不保留凑数。
6. **画嵌套树给用户**：`【大骨架←文件:行号【小骨架=整句母句←文件:行号【换的专名←文件:行号】】】`，证明每一层都有出处、无一自创。**任何一句指不出它的句式母句 = 不能写。**

### 2.3.1 🟥🟥 填词操作纪律（2026-06 科尔段·用户点破的头号操作错误）

**找完骨架后，绝不许凭语感先把成稿写出来、再 grep 验证。** 这样必然大量 0 命中、必然东拼——因为你写出来的词是"想"的，不是"捞"的。

- ❌ **错误操作**：骨架定了 → 凭感觉写"他只是个跑跑腿、递文书的小执法员" → grep → 一片 0 命中 → 再改。本质是"先创作后验证"，改完还是自创。
- ✅ **正确操作**：骨架定了 → 每个空位**先回九本 `grep -hoE ".{0,4}关键词"` 捞出现成搭配** → 从命中结果里挑词填进去。词和短语都是"捞"来的，不是"想"出来的。
- **"填不出词"是伪命题**：只要从 grep 命中里捞，就不可能 0 命中；一旦 0 命中，就是你又"先自己写"了。
- **连两字搭配也要现成**：grep 到"跑腿""打杂"两个单字词，不等于能写"跑腿打杂"——那个组合得 grep 到现成的（否则就是把单字自己拼＝自创）。"挨骂"＋"受气"→"挨骂受气"同理。
- **范例（科尔履历段）**：玩乐508「我只是个…**打工宅男**」→ 空位先 `grep -hoE ".{0,2}打杂.{0,2}"` 捞到现成"打打杂"、`grep -hoE ".{0,2}跑腿的?"` 捞到现成"当跑腿的" → 填成「他只是个打打杂、当跑腿的」。每个空都这样捞，不预先写。

### 2.4 grep 规范（贯穿全程·出过血的）

- **必须 `grep -Hn` 带文件名**，出处书名+行号直接从终端输出抄，**禁止凭印象分配**（否则张冠李戴，被当场抓出"假骨架"）。`grep -h` 合并多本不显示文件名、再靠记忆分书名 = 必然出错。
- **"或"用 `|` 不是 `\|`**（写 `\|` 被当字面竖线，造成大量假 0 命中）。
- **grep 命中 ≠ 语用正确**：看片段在母句里的真实用法/语域是否匹配（「钉在」有命中但母句是"皮被钉在木框"的字面钉，不能拿来写"坐着不动"）。看真实上下文：`grep -hoE "[^，。」「]{0,6}片段[^，。」「]{0,9}" $R`
- **记忆/summary 里的旧母段，用前必 `grep -Hn` 复验**，查不到立即作废重找。
- **停用一世之尊**（仙侠白话会把语感稀释成仙侠味），只可做剧情逻辑参考，不可借句式。

### 2.5 每段 ≤150 字给用户看

附【母句原文 → 套写 → 确切组合 grep -Hn】三件套，控制错误密度，便于随时打断。

完整方法见 [[feedback_skeleton_first]]。

## Step 3 — 调用 novel-rewrite-verify skill

写完一段（或整段）后，写到 `/tmp/check.txt`，调用 `novel-rewrite-verify`。它会跑：所有黑名单 grep（含 cliché 比喻）、短句独段堆砌检测、"立刻命名/按理说"凭空植入检测、反应强度匹配、物理逻辑、伏笔明示检测。

**Why：** 由独立 skill 跑验证，避免"写作的我"做"验证的我"——分工避免认知盲点。

## Step 4 — 按违规清单修复至 0 违规

逐条修复，修复后**再次调用 verify**，直到 0 违规。

## Step 5 — 用户确认后写入

只有用户说"开始/写入/OK 写"之后才能动 Edit/Write。
1. 用 Edit 替换目标段；原文带【要改：…】标记则整块替换含标记。
2. 联动修改一并完成。
3. 写入后用 Read 或 grep 确认。
4. **写入后汇报**：改了哪些行 / 自检最终汇总 / 还有哪些后续可能要改但不在本次范围。

> "Step 0：需求确认（或：需求已明确，跳过）..."

> "Step 2：这是大骨架（母段原文）+ 逐句骨架对照 + grep -Hn 证据..."
