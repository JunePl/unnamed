# 正文写作 / 重写工作流程（98% 贴合度铁律）

**触发条件：** 用户要求修改、重写、新写、续写 `/root/Pi/unnamed/正文/` 目录下的任何小说正文。

**核心铁律：** 每一句话和每一个单词之间的组合都必须能在参考材料、调研报告、以及锚定的两本书（一世之尊 + 术师手册，或场景对应的其他参考）里找到对应组合。**不能 98% 贴合就不写。**

---

## Step 0 — 接到任务先问问题（不可跳过）

**绝不能在没问问题的情况下直接动手。**

按 CLAUDE.md 的项目级铁律——做任何任务前必须先问问题，确保 95% 以上明白用户意思后才能工作。

**必问的问题类型：**
1. **剧情机制**：关键动作的物理/逻辑机制（"米尔顿如何用死换琳莎活命？"）
2. **角色心理动机**：情绪锚点（"含笑去死的内在逻辑是什么？"）
3. **影响范围**：哪段重写、哪段保留、后续是否需要联动修改
4. **AI 浓度判定**：用户说"AI 浓度很高"指的是哪一类问题（句式/视角/凭空编造细节/比喻/超短句/句子只写一半）
5. **角色能力/职业/设定**：写他怎么死/怎么活之前必须知道他能做什么

**问题数量 3-6 个，覆盖能影响写作方向的关键不确定点。**

如果用户给的示例里有反问（"哪里有 X？？？"），意味着原稿里有 AI 凭空编造的细节——必须确认并删除。

## Step 2 — 列每一拍的"想要X但Y所以Z"骨架

写之前必须列出整段动作/事件的拍点，每一拍包含：

| 拍 # | 角色想要做什么 | 阻碍是什么 | 反应/反应是什么 | 视觉/恐怖细节 | 情绪化动词 |
|---|---|---|---|---|---|
| 1 | … | … | … | … | … |

**每一拍至少满足：1 个情绪化动词 + 1 个具体视觉细节。**

情绪化动词词库：迫不得已、不得不、只能、咬牙、狠狠、猛地、直接、立刻、几乎没瞄就、来不及、还没X就、一把、一下子、瞬间。

## Step 3 — 先写 200 字核心样本（绝不直接写完整段）

选最考验语感的那一拍（通常是高强度/情绪峰那段），写 200 字左右。

**视角必须是"角色的工作体验"**，不是"导演画面感"。例如：
- ❌ 导演："柜台后面的傀儡爬过来，舌头伸向琳莎"（俯瞰镜头）
- ✅ 工作体验："阿尔温朝那边冲过去。可他没冲到。"（阿尔温实时感受）

---

## Step 4 — 提交样本时必须附带（缺一不可）

### 4.1 逐句对照表

| 我的句子 | 对照原文 | 应用了什么技法 |
|---|---|---|
| "阿尔温迫不得已先把枪口扣下" | 用户示例："阿尔温迫不得以只能连续两枪将它解决" | "想要X但Y所以Z"挣扎结构 |
| … | … | … |

### 4.2 自检报告（必查项）

#### A. 句式拐杖检查
- 破折号"——"次数：（200字内最多1个）
- "不是X，是Y"句式次数：（一段最多1次）
- "X的那种"贴标签次数：（必须 0）
- 文艺比喻次数：（必须 0，参考标准是"具体物体的功能性比喻 OK"）
- 拟人化怪物（"它认得清X"）：（必须 0）
- 现代词污染（"肌肉记忆""数据""模式"等）：（奥术世界观必须 0）
- 凭空编造细节（V1 里曾有"手提袋"AI 编出）：（必须先确认存在）
- 短句独立成段连续 3+ 次：（不允许）
- 阿尔温情绪反应是否匹配事件稀有度：（避免"脊背发凉""瞳孔一缩"）

#### B. 词级 grep 验证（Ch14上 重写血泪经验，必做）

**核心铁律：每一个动词、每一个修饰副词、每一个句式骨架，都必须能在参考材料里 grep 到对应。0 命中 = 自造，必须替换。**

写完样本/整段后，必须用 Bash 工具对**每一个不寻常的表达**做 grep 验证，命令模板见 §"grep 验证命令模板"。

不能依赖"语感像"或"看起来对"——必须实际跑 grep。

**两类自造的鉴别：**
1. **词级自造**：单个动词/副词参考里没有（"跳到/跌到/坚韧地/挂歪/烫白"等）
2. **句式骨架自造**：单词参考里有，但组合成句的骨架参考里没有（"X在Y上紧了一下"——参考有"握紧X"但没有"X在Y上紧了一下"）

### 4.3 关键不确定点

如果对某一句不确定（比如某个比喻是否过 AI），明说出来让用户判断。

## Step 5 — 根据用户反馈判断错误等级

| 用户说的话 | 错误等级 | 处理方式 |
|---|---|---|
| "差的很远" / "70%" / "完全没写出情绪和紧张感" | **大方向错** | 不改细节。重新检查：视角是不是导演镜头？节奏是不是工作流汇报？是不是缺"想要X但Y所以Z"？ |
| "特别AI" / "这个不行" / "这种是不可以的" | **句式拐杖错** | 找出具体哪类拐杖（"X的那种"/"——"/"不是X是Y"/文艺比喻），全文扫一遍 |
| "这次可以，但是注意X" | **大方向对了，细节错** | 只改 X，保留其他。然后扫全文确认没有同类问题 |
| "完美" / "可以" | **通过** | 进入 Step 7 写入文件 |

**关键教训：第一稿用户说"差的很远"时，不要急着改句子细节。先问视角是否从"角色工作体验"出发，节奏是否"灾难连锁发生"。**

---

## Step 6 — 写完整段时

只有用户对样本说"可以了"或"接着写"之后才能写完整段。

写完整段时：
- 每一句写完都问："这句能在参考里找到对应模板吗？"
- 整段写完后用 Step 4 的自检清单再扫一遍
- 把整段提交给用户**最后确认**（不要直接写入文件）

---

## Step 7 — 写入文件（用户确认后）

只有用户说"开始"/"写入"/"OK 写"之后才能动 Edit/Write 工具。

写入流程：
1. 用 Edit 工具替换目标段（避免整段 Write 覆盖文件）
2. 如果原文带【要改：…】标记，整块替换包括标记本身
3. 联动修改（如"后面的米尔顿都换成琳莎"）必须一并完成
4. 写入后用 Read 工具确认结果，给用户做最终汇报

**写入后的汇报必须包括：**
- 改动了哪些行
- 自检最终汇总
- 还有哪些后续可能要改但不在本次范围内的（比如原文已有的"不是X是Y"句式）

## 红线（违反任何一条都必须重写）

1. ❌ 跳过 Step 0 直接动手
2. ❌ 写之前不读对应调研和参考
3. ❌ 直接写完整段不写 200 字样本
4. ❌ 提交样本不带逐句对照表 + 自检报告
5. ❌ 用户反馈"差的很远"时只改细节不重审视角
6. ❌ 凭空编造细节（手提袋、戒指、信物等没在前文出现过的东西）
7. ❌ 用 AI 拐杖句式（"X的那种"/"不是X是Y"过量/破折号过量/文艺比喻/拟人怪物）
8. ❌ 把"作者觉得酷"硬套成"角色震惊"——情绪强度必须匹配事件稀有度
9. ❌ 没经过用户确认就写入文件

## 附录 B：grep 验证命令模板（写完必跑）

写完样本/整段后，**必须**用 Bash 工具运行以下命令对全文做机械验证。

### B.1 黑名单残留检查（应全部 0）

```bash
# 把目标章节切出来到临时文件
awk 'NR>=START && NR<=END' "/path/to/正文.novel" > /tmp/check.txt

# 黑名单关键词扫描
for w in "跳到\|跳回\|跌到" "再切" "坚韧地" "终极于此" "一格一格" "剥到最里" \
         "挂歪" "扩大锁定" "唇形" "收了一收" "X没顾上" "甜腻味$" \
         "这一线" "腰腹的力" "倒抽一" "借的是动静" "目光锁着" \
         "X熟$" "在闪$" "抠了一下" "这一秒变了" "稳稳地" \
         "脊背发凉" "后背发凉" "呼吸停了" "瞳孔一缩" "心跳漏了" \
         "愣在原地" "喉咙发紧"; do
    c=$(grep -c "$w" /tmp/check.txt 2>/dev/null)
    [ "$c" != "0" ] && echo "残留: $w -> $c次"
done
echo "（无输出=全部0残留）"
```

### B.2 可疑表达逐个 grep 参考材料（验证是否有迹可循）

```bash
REF="/path/to/Reference/术师手册_utf8.txt /path/to/Reference/一世之尊_utf8.txt \
     /path/to/Reference/文风参考.md /path/to/Reference/调研_*.md"

# 例：验证某个动词在参考里是否有
grep -h "倒吸一口" $REF 2>/dev/null | head -3   # 命中 → 可用
grep -h "再切" $REF 2>/dev/null | head -3        # 0 命中 → 自造，替换

# 例：验证某个句式骨架
grep -h "X在Y上紧" $REF 2>/dev/null              # 0 命中 → 必须改"握紧了 Y"
```

### B.3 数量化指标统计

```bash
echo "=== 破折号—— 次数（应≤4，全部为戏剧节点）==="
grep -o "——" /tmp/check.txt | wc -l

echo "=== '不是X，是Y' 标准句式 次数（应≤3）==="
grep -oE "不是[^，。]+，是" /tmp/check.txt | wc -l

echo "=== HUD 数字弹屏【灵值：】 次数（应=0）==="
grep -c "【灵值：" /tmp/check.txt

echo "=== 字符数 ==="
wc -m /tmp/check.txt
```

### B.4 验证流程（强制）

1. **写完样本** → 跑 B.1 黑名单扫描 → 有命中先替换
2. **逐句过 4.1 对照表** → 任何不熟悉的动词/副词/骨架，跑 B.2 单词验证 → 0 命中替换
3. **写完整段** → 再跑 B.1 + B.3 数量化检查
4. **提交用户前** → 把 B.1/B.3 的结果作为自检报告附在样本旁

**不跑 grep 直接交稿是违反 SKILL 的。** 用户每发现一个 AI 自造词，都意味着上面这个流程没做或做得不严。
