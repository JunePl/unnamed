# 正文写作 / 重写工作流程（98% 贴合度铁律 + 三段法）

**触发条件：** 用户要求修改、重写、新写、续写 `/root/Pi/unnamed/正文/` 目录下的任何小说正文。

**核心铁律：** 每一句话和每一个单词之间的组合都必须能在参考材料、调研报告、以及锚定的两本书（一世之尊 + 术师手册，或场景对应的其他参考）里找到对应组合。**不能 98% 贴合就不写。**

**2026 神殿章血泪经验**：单纯的"写完再 grep 自检"不够。**写之前的"节奏分析+骨架搭建"才能从源头避免 AI 默认偏好**。

---

## ⛔ 三条不可绕过的硬验证

### 硬验证 1：每次提交样本必带"已读参考清单"

```
【已读参考】
- 文件 1：[路径] —— 一句话总结这次读到什么有用的句式/语感
- 用户已发布正文 grep 结果：[搜了什么关键词] —— [找到了哪些可借鉴的句子，给出行号]
```

没有这段清单 = 没读 = 不能交稿。

### 硬验证 2：每次提交样本必带"grep 自检日志原始输出"

不是"我跑了，全 0"声明——是终端实际输出。

### 硬验证 3：用户负反馈"差的很远"出现时——禁止局部修补

第一次：5 分钟内重新检查视角/节奏/拍点三件事并向用户汇报。

第二次：禁止继续写样本。停下来让用户口头解释他想要的是什么，**用文字复述给他确认**，确认后再写。

## Step 2.5 — 节奏分析（2026 新增，不可跳过）

**Why：** 之前我"先想内容再凭感觉填句子"——句式骨架是凭感觉写的，所以经常出短独段堆砌/破折号过多/cliché 比喻。**这一步强制把"节奏"作为独立决策**。

### 蓝图必含 4 项

1. **场景类型**：日常对话 / 战斗紧张 / 感官冲击 / 内心独白 / 探索观察 / 高密度信息 / 黑色幽默 / 余韵转场
2. **节奏定调**（参考 [[feedback_skeleton_first]] §B）：
   - 中长句连缀（默认基础——听日/竹宫风格）
   - 戏剧短独段间插（少量，仅 1-2 处揭示节点）
   - 对话密集
   - 挣扎结构（每拍"想要 X 但 Y 所以 Z"）
3. **威胁级别**（如果场景含危险）：L0-L4（参考 [[feedback_scene_consistency]] §B），决定身体反应强度
4. **参考锚点**：明确这段对应**哪个参考作品的哪个场景**——后续骨架/填词都从这个参考找

**输出格式**：500 字以内的节奏蓝图，给用户确认。

### 节奏蓝图示例

> **场景类型**：探索观察 + 致命压感击退
> **节奏定调**：中长句连缀（默认）+ 2-3 处戏剧短独段（揭示节点：发现腿/识别压感=死）
> **威胁级别**：L3（致命压力——身体冻结、不敢动、呼吸停）
> **参考锚点**：
>  - 探索部分 → 默读"鞋很干净"式工作流观察
>  - 致命压感 → 术师手册"喘不过气"+ 用户正文 2994"肺里的气一下子被碾空"
>  - 不能用 → 术师手册的"无形的手攥住" cliché

## Step 3 — 递归骨架（⭐⭐⭐ 整个 skill 最核心的环节，写作成败全在这里）

**这是 skill 里最重要的一步。Step 0-2.5 都为它服务，Step 5 的 verify 主要也是查它。做对这一步 AI 味自然消失；做错，文笔再好也是 AI。** 完整方法见 [[feedback_skeleton_first]]。

### Step 3.0 递归骨架填空法

**核心定义：成稿每一句/每一短语/每一词，都从六本参考 grep 到真实母句"填空换名词"而来，无一自创。** 骨架是递归三层：

- **大骨架** = 整段的真实连续母段，定"由哪些小句、按什么情绪推进与顺序"。
- **小骨架** = 每个小句**各自的一条完整句式母句**。⚠️**这是最易做错的一层**：小骨架是"一整句的句式母句"，**不是几个词**。grep 到"弯腰""捡起""青石板"再拼成"弯腰从青石板上捡起手环"＝错（句式是自创的）；必须找到「(薇瑟)从地上捡起狐狸假面」这样**一整句**，套成"从地上捡起手环"。**找词 ≠ 找小骨架。**
- **词** = 小骨架空位里换进去的专名，也要各自 grep 到确切组合。

**🟥 S 级三铁律（一条都不能破）：**
1. **一句一母句，不准缝合两母句**——母句「回身架起步枪」完整了，不许接「扣下扳机」凑一句；动作链要找本身就完整包含它的母句，否则拆成多个独立小句各对一母句。
2. **不准用孤立词拼无母句结构的句子**——「落了空」+「空无一人」拼一句、「左右两边各站着一个」＝句子结构没母句，禁止。**叙述句容易盯紧，动作/画面流水（弯腰捡起、买面包、群像构图）最易松懈退化成词拼——盯死。** 群像要找「穿着体育服站成一排」这种真实群像母句。
3. **每个确切组合 grep -Hn 到，近义/同词根不算**——「没停」不能拿「没停下」当出处；「那一枪」三本皆无就弃用。换的专名也要各自 grep 到。

**🟥🟥🟥 整句套用铁律（最根本）：成稿 = 一条完整母句，原样保留全部结构/长度/标点/所有非专名的字，只换专有名词。** 三种"借壳"全禁：
- **不许截取母句一部分**：「她融入人群里继续寻找虚实素材」不能只挖"融入人群"丢后半自己接。
- **不许加减/改标点改结构**：「扶着剑袋大口喘气」不能改成「扶着墙，大口喘气」。
- **不许拆词组重组成新句**：「变到再也回不来的地方」不能拆出"再也回不来"重组成「人要是没了就再也回不来了」。
- **不许为塞剧情强改母句的词**：「静静地放下手机」写"放下手环"，别改"记下"。

**判定：想写的意思，参考里找不到一条结构/长度吻合、只换专名的完整整句 → 说明这写法太烂没人会这么写，放弃它，换参考里真实有整句的表达，绝不截取/改标点/拆词硬凑。**

**✅ 但别教条到瘫痪——保留完整结构的前提下，可换专名/代词、为通顺做近义动词微调。** 范例：术师15690「剑姬魔女看见石头在观者手里忽然化成齑粉，顺着他的指间滑落下来」→「雷纳德看着手环在自己手里忽然化成齑粉，顺着他的指间滑落下来」（换主语+近义动词不算截取，结构没动＝合格）。

**操作**：①定效果 ②找大骨架(真实连续母段) ③**逐个小句找一条完整句式母句(小骨架)** ④母句不足则把该小句当新骨架下钻 ⑤逐组合 grep -Hn 复核(含语域) ⑥画嵌套树给用户：`【大骨架←文件:行号【小骨架=整句母句←文件:行号【换的专名←文件:行号】】】`。**任何一句指不出它的句式母句 = 不能写。**

### Step 3.1 grep 规范（贯穿全程）

- **参考池用六本**：`R="Project/Reference/终局委员会.txt Project/Reference/玩乐关系.txt Project/Reference/术师手册_utf8.txt Project/Reference/弑神者.txt Project/Reference/骑士之潭.txt Project/Reference/魔战公主.txt"`（终局/玩乐/术师 + 新增弑神者/骑士之潭/魔战公主，后三本无 _utf8 后缀；停用一世之尊；）。
- **必须 `grep -Hn` 带文件名**，出处书名+行号直接从终端输出抄，**禁止凭印象分配**（否则张冠李戴，被用户当场抓出"假骨架"）。
- **"或"用 `|` 不是 `\|`**（写 `\|` 造成大量假 0 命中）。
- **grep 命中 ≠ 语用正确**：看片段在母句里的真实用法/语域是否匹配（「钉在」有命中但母句是"皮被钉在木框"的字面钉，不能拿来写"坐着不动"）。`grep -hoE "[^，。」「]{0,6}片段[^，。」「]{0,9}" $R`
- **记忆里的旧母段用前必 `grep -Hn` 复验**，查不到立即作废重找。

详见 [[feedback_skeleton_first]]。

## Step 4 — 逐组合复核 + 分段给用户

1. **成稿后每个短语回 `grep -Hn` 复核确切组合**（近义/同词根不算）——任一处查不到 → 改写迁就母句，不许保留凑数。
2. **每段 ≤150 字立刻给用户看**，附【母句原文 → 套写 → 确切组合 grep -Hn】三件套，控制错误密度。
3. **前文已用术语严格继承**：grep 用户已发布正文，不改说法（用户写"灵纹"不能写"蛛印"）。

详见 [[feedback_skeleton_first]]。

## Step 5 — 调用 novel-rewrite-verify skill（2026 新增）

**Why：** 由独立 skill 跑验证，避免"写作的我"做"验证的我"——分工避免认知盲点。

写完一段（或整段）后：

```
调用 Skill: novel-rewrite-verify
输入: 刚写的段落（写到 /tmp/check.txt）
处理: 自动跑所有 feedback 检查
输出: 违规清单 + 修改建议
```

`novel-rewrite-verify` 会跑：
- 所有黑名单 grep（包括 2026 新增的 cliché 比喻）
- 短句独段堆砌检测
- "立刻命名"/"按理说"等凭空植入检测
- 反应强度匹配检测
- 物理逻辑检测
- 伏笔明示检测

---

## Step 6 — 根据验证结果修复

根据 verify 输出的违规清单**逐条**修复。修复后**再次调用 verify** 直到 0 违规。

---

## Step 7 — 用户确认后写入

只有用户说"开始"/"写入"/"OK 写"之后才能动 Edit/Write。

**写入流程：**
1. 用 Edit 工具替换目标段
2. 如果原文带【要改：…】标记，整块替换包括标记本身
3. 联动修改必须一并完成
4. 写入后用 Read 工具或 grep 确认结果

**写入后汇报必须包括：**
- 改动了哪些行
- 自检最终汇总
- 还有哪些后续可能要改但不在本次范围内的

## 红线（违反任何一条都必须重写）

1. ❌ 跳过 Step 0 直接动手
2. ❌ 跳过 Step 2.5 节奏分析
3. ❌ 违反 S 级三铁律（**最严重**）——①缝合两母句 ②用孤立词拼无母句结构的句子 ③拿近义/同词根冒充确切组合出处 ④没母句就凭空组织句子/推进逻辑。任一条都是 AI 味总根源
4. ❌ 写之前不读对应调研和参考
5. ❌ 直接写完整段不分小段给用户看
6. ❌ 提交样本不带逐句对照表 + 自检报告
7. ❌ 不调用 novel-rewrite-verify skill 就交稿
8. ❌ 用户反馈"差的很远"时只改细节不重审视角
9. ❌ 凭空编造细节（手提袋、戒指、信物等没在前文出现过的东西）
10. ❌ 用 AI 拐杖句式（详见所有 feedback_*.md）
11. ❌ 把"作者觉得酷"硬套成"角色震惊"——情绪强度必须匹配事件稀有度

13. ❌ 自造前文已建立的术语的替代说法（如用户写"灵纹"我写"蛛印"）
14. ❌ 已知类型事物的"第一眼识别"也立刻命名（详见 [[feedback_human_cognition_process]] §E）
