# 读者视角设定文档 · 说明（先读这个）

## 用途
本文件夹维护一份**严格的"读者视角"**设定与剧情记录，目的只有一个：**确保后续正文推进不出纰漏**——不提前泄露伏笔、不制造信息错位、不与已明写的设定冲突。

writer / 作者掌握大量上帝视角信息（伏笔答案、角色真实身份、未揭示的真相）。本文档**刻意剔除**这些，只保留**一个只读过正文的读者会知道、以及能凭正文合理推断的**。写新章前对照本档，就能站在读者的认知水位上校准。

## 铁律
1. **只采信正文白纸黑字**（`正文/正文.novel` 第 1–21 章）。正文没写的，一律不写。
2. **可以合理推断，但必须是读者凭正文就能推出的**（如"阿尔温和阿涅丝是兄妹"——正文明写）。推断项一律标注 **【推断】**，并附推断依据章节。
3. **严禁写入任何伏笔答案 / 上帝视角真相**，哪怕作者 / writer 知道。判断标准：这条信息若正文没明写、需要"作者告诉你"才知道，就是伏笔，**不写**。
   - ✅ 写："城里近两年黑猫变多，有'黑猫蹲谁家门口、谁就出事'的传说"（正文明写的现象）
   - ❌ 不写："黑猫是某某人的宠物 / 受某人操控"（正文未写＝伏笔）
   - ✅ 写："街口有个瘦高、灰白短发、眼神很冷的流浪汉，阿涅丝莫名觉得他不对劲，后来他不见了"（读者看到的现象）
   - ❌ 不写："那个流浪汉就是雷纳德 / 和北境列车那个杀手有关"（伏笔）
4. **伏笔只记现象、归入"读者悬念清单"，不记答案**。读者好奇但还不知道答案的，列进 `剧情时间线.md` 的"读者悬念"区——只写"读者会好奇 X"，绝不写 X 的真相。
5. **设定数字 / 专名也守此线**：作者口中的设定（如某体系的确切数量、某教派的本质）若正文没明写，不写确切值，只写正文给出的部分（如"提耶林是离王都较近的州城，西边另有更远的州城"，而非"全国共九个州城"——除非正文真写了九个）。

## 文件
- `人物.md` — 读者视角人物档案（外貌 / 身份 / 性格 / 关系 / 能力；明写 + 【推断】）
- `世界设定.md` — 读者视角世界观（地理 / 灵使体系 / 社会 / 机构 / 物件 / 时间线）
- `剧情时间线.md` — 按章剧情推进 + **读者悬念清单**（只列悬念，不写答案）

## 维护
**每次推进正文后，必须回来按新章节的读者视角增补本档**（新人物、新设定、新事件、新悬念）。此规则已写入 `novel-rewrite` skill 与 writer memory。
